## Project Overview

MealPrepFriend is an AI-powered weekly meal prep planner. Users set calorie/macro goals, select food preferences, and receive a 7-day meal plan generated by Google Gemini (via Supabase Edge Functions). The app also generates a grocery list and batch cook guide.

**Live product:** Full-stack React SPA deployed on Netlify, backed by Supabase (PostgreSQL + Auth + Edge Functions).

## State Management

**MealPrepContext** (`src/context/MealPrepContext.tsx`) is the single source of truth for:
- User profile: biometrics, calorie/macro targets
- Food preferences: protein/carb/veggie/fat choices, cuisines, complexity
- Generated data: meal plan, grocery list, cook guide
- Onboarding state

Data is persisted to Supabase `profiles` table. The context loads from Supabase on mount and writes back on save.

**Pattern for updates:**
1. User fills form on a page
2. Page calls a context method (e.g., `updateProfile`, `setMealPlan`)
3. Context updates local state and writes to Supabase
4. Downstream pages read from context

## Database Schema

### `public.profiles`
Stores all user data. One row per user (linked to `auth.users`).

Key columns:
- Biometrics: `biological_sex`, `age`, `height_ft`, `height_in`, `height_cm`, `weight_lbs`, `weight_kg`, `bmi`, `bmr`, `tdee`, `activity_level`
- Goals: `goal` (weight_loss/gain/maintain), `target_calories`, `protein_target`, `carb_target`, `fat_target`
- Food preferences: `protein_choice`, `carb_choice`, `veggie_choice`, `fat_choice`, `food_avoidances`, `cuisine_preferences`, `complexity_level`, `biggest_meal`, `healthy_swaps_enabled`
- Settings: `unit_preference`, `theme_preference`, `onboarding_completed`, `household_size`

### `public.vault_meals`
Saved meals per user.

Key columns: `meal_name`, `calories`, `protein_g`, `carb_g`, `fat_g`, `prep_time_min`, `is_staple` (max 3 staples per user)

**RLS is enabled on all tables.** Every query must be performed in an authenticated context using the Supabase client.

## Edge Functions (AI Integration)

Located in `supabase/functions/`. Written in Deno TypeScript.

### `generate-meal-plan`
- Accepts user preferences + calorie/macro targets + locked meals + staple meals
- Calls Lovable AI Gateway → Gemini for structured JSON output
- Returns 7 `DayPlan` objects with meals, macros, and timing
- Accuracy requirement: ±75 cal of daily target

### `generate-grocery-list`
- Accepts a meal plan
- Returns items grouped by category (Proteins, Produce, Grains, Dairy, Pantry)
- Includes quantities, units, and estimated USD prices

**Calling edge functions from the frontend:**
```ts
const { data } = await supabase.functions.invoke('generate-meal-plan', {
  body: { ...payload }
})
```

**Secrets needed in Supabase:** `LOVABLE_API_KEY`

## Development Workflow

### Setup
```bash
npm install
# Create .env with Supabase credentials
npm run dev       # Starts on http://localhost:8080
```

### Key Commands
```bash
npm run dev           # Dev server (port 8080)
npm run build         # Production build → dist/
npm run preview       # Preview production build
npm run lint          # ESLint check
npm run test          # Run tests once
npm run test:watch    # Tests in watch mode
```

### Adding a New Page
1. Create `src/pages/NewPage.tsx`
2. Add route in `src/App.tsx` (wrap with `<ProtectedRoute>` if auth required)
3. Add navigation link to `BottomTabBar.tsx` and/or `FloatingTabBar.tsx` if needed

### Adding a Database Column
1. Create a new migration file in `supabase/migrations/` (never edit existing migrations)
2. File name format: `YYYYMMDDHHMMSS_description.sql`
3. Update `src/integrations/supabase/types.ts` to reflect the new column
4. Update `MealPrepContext.tsx` if the column needs to be in global state

### Adding an Edge Function
1. Create `supabase/functions/<function-name>/index.ts`
2. Follow the Deno + CORS pattern from existing functions
3. Access via `supabase.functions.invoke('<function-name>', { body: ... })`

## Common Pitfalls

1. **Do not edit `src/components/ui/`** directly. These are managed by shadcn/ui.
2. **Do not edit existing Supabase migrations.** Always add a new file.
3. **Do not edit `src/integrations/supabase/types.ts`** manually — regenerate with Supabase CLI.
4. **Do not commit `.env`** — it is in `.gitignore`.
5. **RLS is enabled.** Database queries will silently return empty results if run without an authenticated session.
6. **Edge functions use Deno**, not Node.js. Do not use Node-specific APIs or npm imports inside `supabase/functions/`.
7. **Meal plan calorie target tolerance is ±75 cal/day** — respect this constraint in AI prompts.
8. **Max 3 staple meals** per user in the vault (`is_staple` flag).
